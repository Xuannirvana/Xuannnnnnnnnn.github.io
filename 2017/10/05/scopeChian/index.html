作用域和作用域链一直是 JavaScript 中比较难弄明白的两个概念，除了它们自身的概念比较繁琐复杂外，还有一个原因就是因为它们还涉及到了很多其他零零散散的知识点。所以在学习作用域与作用域链之前，需要先弄明白我们博客前面几章讲的概念。

作用域链

很多人都认为作用域链跟作用域有很大的关系，所以每次学习这两个知识点的时候都是先学习作用域，再去学习作用域链。

但是我们这次，先讲作用域链。

我们知道，当函数被调用激活时，会开始创建对应的执行上下文。在执行上下文生成的过程中，变量对象、作用域链、以及this的值会分别被确定。在前面我们已经详细讲解了关于变量对象的分析，所以这里，我们主要来分析一下作用域链。

作用域链，是由当前上下文与上层上下文中的一系列变量对象组成，它保证了当前执行环境对符合访问权限的变量和函数的有序访问。

作用域

在讲解作用域这个概念之前，我们需要先了解另外两个东西：一个叫做编译器，一个叫做引擎。

编译器和引擎

在前面我们讲《变量提升和函数提升》这一章的时候，就已经知道代码的执行过程实际上大致可以分为两个阶段：

  • 代码编译阶段：这个阶段，内部代码并未开始执行，只是将代码编译成了可执行的代码。

    • 函数提升
    • 变量提升
    • … …

  • 代码执行阶段：这个阶段，代码开始执行，即执行第一步中编译完成的可执行代码。

    • 变量赋值
    • 函数引用
    • … …

而完成这两个阶段工作的，就是我们的编译器和引擎。

  • 编译器：负责语法分析及代码生成等工作。
  • 引擎：负责整个 JavaScript 程序的执行过程。

作用域概念

我们先来看“作用域”这三个字是什么意思。我们把“作用”和“域”拆开来看。

  • 作用：访问、操作、调用…
  • 域：区域、范围、空间…

因此，作用域就是变量和函数的可访问范围，或者说变量或函数起作用的区域。

那么有了这个作用域，到底有什么作用呢？

不同的作用域可以限制变量或函数不同的作用范围，因此，它可以用来管理编译器在工作时如何去对变量和函数进行查找和访问。

作用域分类

首先明确一个概念，JavaScript 中的作用域是通过函数取划分的，所以不存在块级作用域。

[ 注：这里我们讲的只针对 ES5。ES6中，因为加入了let关键字，用let声明的变量具有块级作用域。]

JavaScript 中的作用域只有两种（因为eval我们平时开发中几乎不会用到，所以这里不考虑）：

  • 全局作用域
  • 局部（函数）作用域

全局作用域中的所有的变量和函数可以在代码的任何地方访问和使用，而局部作用域中的所有的变量和函数只能在当前作用域内部使用。

作用域与执行上下文

我们已经分析讲解完了作用域和执行上下文这两个知识点，也基本知道作用域和执行上下文各自的职责，可以看出它们几乎是没有什么交集的。那么为什么很多人还是会把这两者弄混呢？它们两个之间到底什么联系和区别呢。

联系

在一个函数被执行时，创建的执行上下文对象除了保存了些代码执行的信息，还会把当前的作用域保存在执行上下文中。所以它们的关系只是存储关系。

区别

  • 产生时间的区别：

    作用域的范围在函数声明时确定。

    执行上下文在函数调用时创建。

  • 作用的区别：

    作用域的作用是用来控制变量和函数的作用范围。

    执行上下文的作用是用于跟踪代码的运行情况。
