作用域与作用域链

作用域和作用域链一直是 JavaScript 中比较难弄明白的两个概念，除了它们自身的概念比较繁琐复杂外，还有一个原因就是因为它们还涉及到了很多其他零零散散的知识点。所以在学习作用域与作用域链之前，需要先弄清楚我们前面几章讲的概念。

编译器和引擎

在讲解作用域这个概念之前，我们需要先了解另外两个东西：一个叫做编译器，一个叫做引擎。

在前面我们讲《变量提升和函数提升》这一章的时候，就已经知道整个代码的执行过程实际上分为了两个阶段：

  • 代码编译阶段：即将代码翻译成可执行代码，其中就包含了我们的变量提升和函数提升这两个过程。
  • 代码执行阶段：即执行第一步中翻译完成的可执行代码。

而完成这两步工作的，就是我们的编译器和引擎。

  • 编译器：负责语法分析及代码生成等工作。
  • 引擎：负责整个 JS 程序的执行过程。

作用域

了解了编译器和引擎之后，接下来我们开始来看一下什么是作用域。

“域”，我们一般都理解为“区域”，也就是“范围”。那么，作用域，我们也就可以理解为“作用区域”或“作用范围”。

而我们的编译器在工作的时候，操作的无非也就是变量和函数这两样东西了。因此：

作用域，指的就是变量或函数的作用范围。

那么有了这个作用域，到底有什么作用呢？

不同的作用域可以限制变量或函数不同的作用范围，因此，它可以用来管理编译器在工作时如何去对变量和函数进行查找和访问。

分类

首先明确一个概念，JavaScript 中的作用域是通过函数取划分的，所以不存在块级作用域。

[ 注：这里我们讲的只针对 ES5。ES6中，因为加入了let关键字，用let声明的变量具有块级作用域。]

JavaScript 中只有两种（因为eval我们平时开发中几乎不会用到，所以这里不考虑）：

  • 全局作用域
  • 局部（函数）作用域

全局作用域中的所有的变量和函数可以在代码的任何地方访问和使用，而局部作用域中的所有的变量和函数只能在当前作用域内部使用。

与执行上下文的区别

作用域与执行上下文是两个完全不同的概念。但是很多人在这两个概念上都很容易弄混淆，所以一定要仔细区分。

关于作用域的分析讲解就到此结束，接下来我们来看一下 —— 作用域链。

作用域链，是由当前上下文与上层上下文中的一系列变量对象组成，它保证了当前执行环境对符合访问权限的变量和函数的有序访问。
