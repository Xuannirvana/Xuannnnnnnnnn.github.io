在 JavaScript 中，我们不可避免的需要声明变量和函数，可是 JavaScript 编译器是如何找到这些变量的呢？所以我们还需要对执行上下文有一个进一步的了解。

执行上下文生命周期

在前面《执行上下文（Execution Context）》中我们已经知道，当调用一个函数时，一个新的执行上下文就会被创建。但前面没有讲到的是，一个执行上下文的生命周期是可以分为两个阶段的。

  • 创建阶段

    在这个阶段中，执行上下文内会分别创建变量对象(VO)、建立作用域链(SC)，以及确定this的指向。

  • 代码执行阶段

    创建阶段完成后，就会开始执行代码。这个时候，会完成变量赋值、函数引用，以及执行其他代码。

到这里我们就可以看出来详细了解执行上下文有多重要了。这里面涉及的变量对象、作用域链、this等等，恰好都是很多人一直弄不明白，但又极为重要的概念。所以这些概念我们后面都会一一详细总结，而这篇博客我们先重点了解“变量对象（VO）”。

变量对象

变量对象，Variable Object，简称 VO。用来记录在当前执行环境中可以访问到的变量。

变量对象创建过程

变量对象的创建，依次经历了以下几个过程：

  1. 建立arguments对象：检查当前上下文中的函数参数，建立该对象下的属性名与属性值。

  2. 检查当前上下文的声明式函数：在变量对象中以函数名建立一个属性，属性值为指向该函数所在的内存地址的引用。如果函数名的属性已经存在，那么该属性将会被新的引用所覆盖。

  3. 检查当前上下文中var的变量声明：chu每找到一个变量声明，就在变量对象中以变量名建立一个属性，属性值为undefined。如果该变量名的属性已经存在，为了防止同名的函数被修改为undefined，则会直接跳过，原属性值不会被修改。

全局上下文的变量对象

以浏览器为例，全局对象为 window。

全局上下文有一个特殊的地方，它的变量对象，就是 window 对象。而这个特殊，在this的指向上也同样适用，this也是指向 window。

除此之外，全局上下文的生命周期，与程序的生命周期一致，只要程序运行不结束（例如关闭浏览器），全局上下文就会一直存在。其他所有的上下文环境，都能直接访问全局上下文的属性。
